## Problem Statement

Financial institutions process thousands of digital transactions every day across multiple banks, payment methods, merchants, cities, and customer demographics. Analyzing this large volume of transactional data manually makes it difficult to monitor customer spending behavior, account balances, and transaction trends.

This Power BI dashboard provides an interactive analytics solution that enables users to monitor monthly transaction amounts, remaining account balances, customer demographics, merchant-wise transactions, and payment behavior. Dynamic filters and visualization switching allow stakeholders to explore banking insights efficiently and support data-driven decision making.

---

## Steps Followed

### Step 1: Data Collection
- Imported the banking transaction dataset into Power BI Desktop.
- Verified data quality and identified missing or inconsistent values.

### Step 2: Data Cleaning & Transformation
- Performed data cleaning using Power Query.
- Corrected data types and formatted date fields.
- Removed duplicate and unnecessary records.
- Created Month and Age Group columns for analysis.

### Step 3: Data Modeling
- Built relationships between transaction and lookup tables.
- Designed an optimized data model for efficient reporting.

### Step 4: DAX Calculations
- Created DAX measures for:
  - Total Transaction Amount
  - Remaining Balance
  - Transaction Count
  - Monthly Amount
  - Monthly Balance

### Step 5: Dashboard Development
Developed interactive dashboard pages featuring:
- Dynamic column and line charts
- Matrix report with conditional formatting
- Multiple slicers for interactive filtering
- Monthly balance analysis
- Transaction amount trends
- City-wise balance comparison

### Step 6: Interactive Features
Implemented:
- Bookmark navigation
- Button-based chart switching
- Cross-filtering across visuals
- Dynamic slicers
- Conditional formatting
